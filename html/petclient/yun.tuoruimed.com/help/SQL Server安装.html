SQL Server安装

1、使用安装包镜像文件“cn_sql_server_2008_r2_enterprise_x86_x64_ia64_dvd_522233.iso”， 将安装文件刻录成光盘或者用虚拟光驱加载，或者直接解压，打开安装包文件夹，找到并 双击安装源程序“setup.exe”，解压安装文件如下图1-1-1所示：

图1-1-1

2、在左侧的目录树中选择"安装”。在右侧的选择项中，选择第1项目“全新安装或向现有安装添加功能”，然后就进入了安装程序，如下图1-1-2所示：

图1-1-2

3、安装程序支持规则,安装向导运行规则检查，规则必须全部通过后，如下图，单击【确定】进入下一步操作界面

4、进行到此步骤如果没有产品序密钥可以选择安装评估版（180天到期）和免费的Express版本，输入产品密钥后单击【下一步】

5、单击选择“我接受许可协议”再继续点击【下一步】进入安装程序支持的文件

6、安装程序支持文件

7、接下来是安装程序支持规则的检测，通过之后继续【下一步】，如下图有一个Windows防火墙的警告，忽略即可

8、设置安装的功能，选择SQL Sever功能安装

9、在功能选择界面单击选中需要的功能组件，或者全选，单击【下一步】

10、安装规则检查，通过后点击【下一步】

11、在实例配置界面的已安装的实例列表可以看到当前是否存在已安装的实例。如果当前已经存在一个默认实例，此种情况如果选择默认实例进行安装将在下方信息栏将出现错误提示信息而安装成功，错误提示信息如下所示：

实例ID不能包含空格或特殊字符。

该实例名称已在使用。

当前操作系统如果初次安装SQL程序可以选择“ 默认实例 ”进行安装，如果已经存在一个或多个实例，那么只能选择命名实例进行安装，输入自定义的实例名（实例名必须符合规范并且不能与已存在的实例名重复），再单击【下一步】进行安装。

12、接下来安装向导将根据之前的选项确定需占用的磁盘空间，一般需占用7000MB的磁盘空间，确定所选目录空闲空间足够则继续单击【下一步】

13、接下来是定义成功安装后，服务器上SQL Server服务对应的启动账户。可以通过单击“对所有SQL Server服务使用相同的账户”按钮，统一指定SQL Server服务的启动账户。

14、在数据库引擎配置界面可以为数据引擎指定身份验证模式和管理员。从数据库服务器的安全角度考虑建议使用混合模式，使用混合模式验证必须指定内置的sa账户密码，此密码必须符合SQL定义的强密码策略。选择身份验证模式：混合模式(SQL Server身份验证和Windows身份验证)，并设置管理员账户（sa）的账户密码

15、点击【添加当前用户】，指定SQL Server管理员为操作系统登录账号，点击【下一步】

16、在Reporting Services配置界面，一般建议选择“安装本机模式默认配置”选项进行安装，点击【下一步】

17、接下来是错误报告的情况，如果不想将错误报告发送给Microsoft可以不选择任何选项，直接单击【下一步】

18、在安装规则界面，安装程序自动运行检测程序，当列表中的规则检测通过之后再单击【下一步】

19、如下图所示的准备安装界面，列示出了之前所做的设置以供检查，如果还有待修改之选项可以单击【上一步】，返回修改。如检查无误则点击【安装】，开始安装

20、如下图所示的安装进度界面可以查看到安装进度及安装结果报告

21、最后显示安装结果，在此界面提供了日志文件以供分析失败原因。安装完成系统将提示需要重新启动方可正常工作
